    信息系统安全概述

    掌握目标

    了解信息系统的主要安全威胁

    • 掌握信息安全的基本属性（机密性、完整性、可用性）及其含义
    • 信息安全模型

    核心知识

    信息系统安全的基本属性：

    • 机密性：确保信息不被非授权者获得与使用。例如： 信息泄露， 例如密码， 文件等
    • 完整性：信息是真实可信的， 其发布者不被冒充，来源不被伪造， 内容不被篡改。例如： 身份伪造、 文件篡改
    • 可用性：保证信息可被授权人在需要时立即获得并正常使用。例如： DDOS、 勒索软件等

    核心知识

    恶意软件的分类：

    • 计算机病毒
    • 木马
    • 后门
    • Rootkit
    • 勒索软件

    程序在内存中的分布：

    栈溢出：是一种特定类型的缓冲区溢出，它发生在程序使用栈（stack）来管理函数调用和局部变量的内存时。为了实现栈溢出，要满足两个条件。第一，程序要有向栈内写入数据的行为；第二，程序并不限制写入数据的长度。

    栈溢出攻击：需要在溢出数据内包含攻击指令的内容或地址，并且要将程序控制权交给该指令。攻击指令可以是自定义的指令片段，也可以利用系统内已有的函数及指令。攻击的目标就是让 eip 载入攻击指令的地址。

    三个核心寄存器：

    • esp 用来存储函数调用栈的栈顶地址，在压栈和退栈时发生变化
    • ebp 用来存储当前函数状态的基地址，可以用来索引确定函数参数或局部变量的位置
    • eip 用来存储即将执行的程序指令的地址，cpu 依照 eip 的存储内容读取指令并执行

    程序对栈的使用：

    这里介绍一下callee结束后的流程：（重要）

    1. 局部变量会从栈内直接弹出，栈顶会指向被调用函数（callee）的基地址
    2. 调用函数（caller）的基地址从栈内弹出，并存到 ebp 寄存器内
    3. 将返回地址从栈内弹出，并存到 eip 寄存器内

    shellcode：在溢出数据内包含一段攻击指令，用攻击指令的起始地址覆盖掉返回地址。攻击指令一般都是用来打开 shell，从而可以获得当前进程的控制权，所以这类指令片段也被成为“shellcode”。

    shellcode的局限：

    1. 系统关闭内存布局随机化 (ASLR) ，才能确定 shellcode 的起始地址
    2. 函数调用栈具有可执行权限
    3. 系统无栈破坏检测（一个函数对应一个stack frame，每个stack frame都需要一个canary）

    核心知识

    模糊测试（ Fuzz testing） 是一种发现安全漏洞的有效的动态测试方法， 通过向目标系统提供非预期的输入并监视异常结果来发现软件漏洞。

    程序插桩： 借助往被测程序中插入操作， 来实现测试目的的方法， 即向源程序中添加一些语句， 实现对程序语句的执行、 变量的变化等情况进行检查。

    用户身份认证机制

    掌握目标

    掌握身份认证原理及机制

    • 身份认证方法
    • Kerberos

    参考资料：2022年软考信息安全工程师考试必考知识点_kerberos

    核心知识

    三种方法验证主体身份：

    • 只有该主体了解的秘密， 如口令、 密钥
    • 主体携带的物品， 如智能卡和令牌卡
    • 只有该主体具有的独一无二的特征或能力， 如指纹、 声音、 视网膜图或签字等

    重点的身份认证的方式：

    • 口令认证：最简单、 最普遍的身份识别技术。口令是相互约定的代码，只有用户和系统知道。

    口令加盐：是一种安全技术，通常用于存储用户的密码或敏感信息。每个用户的密码都会与一个唯一的随机字符串（盐）相结合，然后再进行哈希处理。使用盐后，相同的密码在不同用户之间也会产生不同的哈希值，注意每个用户的盐都是唯一的。这样做可以防止彩虹表攻击，提高密码强度。

    • 局域网身份认证-Kerberos：属于网络认证协议，其目标是使用密钥加密为客户端/服务器应用程序提供强身份认证。其技术原理是利用对称密码技术(DES加密算法)，使用可信的第三方来为应用服务器提供认证服务，并在用户和服务器之间建立安全信道。

    Kerberos工作流程图：

    Kerberos中四个重要的实体：

    1. Kerberos客户机，用户用来访问服务器设备：
    2. AS(Authentication Server,认证服务器)，识别用户身份并提供TGS会话密钥；
    3. TGS(Ticket Granting Server,票据发放服务器)，为申请服务的用户授予票据(Ticket);

      票据(Ticket)是用于安全的传递用户身份所需要的信息的集合，主要包括客户方实体名称、地址、时间戳、票据生存期和会话密钥等内容

    4. 应用服务器(Application Server),，为用户提供服务的设备或系统。

    核心知识

    • 自主访问控制（ DAC， Discretionary Access Control）

    配置粒度小但是工作量大效率低

    主要包括下列形式：

    • 访问控制表ACL(Access Control Lists)：每个客体附加一个它可以访问的主体的明细表。
    • 访问能力表(Capabilities)：每个主体都附加一个该主体可以访问的客体的明细表。
    • 访问控制矩阵
    • 授权关系表

    优点：基于主体，具有高度灵活性，可扩展性，动态的，细粒度

    缺点：没有对已经具有权限的主体如何使用和传播信息强加任何限制

    用于商业、工业领域，主流的操作系统(Unix、Windows)，防火墙ACL

    • 强制访问控制（ MAC， Mandatory Access Control）

    配置粒度大但是缺乏灵活性，常用于军事

    其访问控制关系分为：上读下写（完整性）、下读上写（机密性）

    参考资料：可信计算概述

    参考资料：可信计算笔记（1）

    核心知识

    • 基础知识

    可信计算（Trusted Computing，简称TC）是一项由TCG(可信计算组)推动和开发的技术。可信的核心目标之一是保证系统和应用的完整性，从而确定系统或软件运行在设计目标期望的可信状态。可信主要通过度量和验证的技术手段实现。度量就是采集所检测的软件或系统的状态，验证是将度量结果和参考值比对看是否一致，如果一致表示验证通过。

    静态度量通常指在运行环境初装或重启时对其镜像的度量。动态度量和验证指在系统运行时动态获取其运行特征，根据规则或模型分析判断系统是否运行正常。

    可信计算技术的四大板块：信任根（RTM RTR RTS）、信任链（度量存储报告机制）、可信平台模块（TCG的TPM、中国的TCM和TPCM）、可信支撑软件（TSS）

    • 可信度量根RTM、可信存储根RTS和可信报告根RTR

    TCG定义的信任根包括3个根。可信度量根（RTM）：负责完整性度量；可信报告根（RTR）：负责报告信任根；可信存储根（RTS）：负责存储信任根。其中，RTM是一个软件模块、RTR是由TPM的平台配置寄存器（PCR）和背书密钥（EK）组成、RTS是由TPM的PCR和存储根密钥（SRK）组成。

    RTM在构建信任链的过程中，将完整性度量形成的信息传递给RTS，RTS使用TPM的平台配置寄存器存放度量扩展值、使用TPM提供的密码学服务保护度量日志。RTR主要用于远程证明过程，向实体提供平台可信状态信息，主要内容包括平台配置信息、审计日志、身份密钥（一般由背书密钥或者基于背书密钥保护的身份密钥承担）。

    • 可信启动的实现原理

    系统从一个小的“信任根（CRTM）”启动，“信任根”是系统中第一个获得控制权的模块；

    信任根启动BIOS；

    BIOS启动引导程序；

    引导程序加载系统内核；

    系统内核启动系统的应用层程序。

    平台配置寄存器PCR 采用扩展的方式记录系统启动序列。由于TPM没有足够大的内存去记录整个操作系统内核，TPM利用计算摘要（Hash）的方式将系统序列中的不同镜像的度量值（Hash 值）存入PCR，存入的过程被称为“扩展”。
