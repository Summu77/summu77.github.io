    数据链路层

    内容主要基于武大网安学院计算机网络课程教学PPT（LYQ老师）

    部分内容参考CHATGPT3.5和CHATGPT4的回答及互联网资源，用于更好的解释

    结构根据教学大纲和PPT自行梳理概括

    数据链路层基础知识

    • 掌握数据链路的概念、数据链路层的功能和实现位置
    • 掌握成帧、透明传输的概念与方法
    • 掌握差错控制的原理，了解循环冗余校验

    基础概念

    数据链路由链路及控制数据在物理线路上传输的硬件和软件共同构成——逻辑链路。

    当采用复用技术时，一条链路上可以有多条数据链路

    数据链路层实现的位置：网络接口卡（NIC）俗称网卡（Network Card），也被称为网络适配器（Network Adapter）

    数据链路层主要的作用：

    • 通过一些数据链路层协议，在不太可靠的物理链路上实现可靠的数据传输。
    • 屏蔽物理层的细节，为网络层提供设计良好的服务接口

    数据链路的两种类型：

    • 点对点信道：只能一对一通信，使用点对点数据链路层通信协议，如PPP、HDLC，主要用于广域网

    • 广播式信道：可以一对多通信，使用MAC（Media Access Control）协议，且共享信道时需要解决碰撞问题，如以太网的CSMA/CD、WLAN的CSMA/CA，主要用于局域网

    数据链路层的功能：

    概括一下：

    • 为网络层提供设计良好的服务接口
    • 封装数据帧（包括帧结构定义、帧定界和帧的透明传输问题）
    • 处理传输差错
    • 调整帧的流速

    封装成帧与透明传输

    • 字符界定法
      • 使得网络传输依赖于8位字符，特别是ASCII字符。
      • 如果转义字符也出现在数据当中，那么应在转义字符前面插入一个转义字符
    • 位串界定法
      • 使用如下的位串作为开始和结束标志：“01111110”
      • 发送方遇到数据中有五个连续“1”时，自动在其后插入“0”；接收方将“0” 取出丢弃。
      • 优点：允许数据帧中包含任意个数的比特，而不会象字符填充法依赖于8位字符。

    数据链路层的透明传输指的是在数据链路层中传输的数据帧对上层协议（如网络层）来说是透明的，即上层协议不需要关心底层数据链路层的具体实现细节。

    差错控制

    差错控制的两种基本策略：

    • 前向纠错：在每一个要发送的数据块上附加足够的冗余信息（纠错码），使接收方能够推导出正确的字符应该是什么。—— 开销大，目前网络协议未用
    • 检错重发：只加入适当的冗余位（检错码），使接收方只知道有差错发生，但不知道是什么样的差错，然后让接收方请求重传。——在计算机通信中常用，普遍使用CRC（Cyclic Redundancy Check，循环冗余检验）算法

    CRC循环冗余检验/多项式校验

    其实可以理解为除法异或运算

    注意：接收端余数为0，并不一定没有差错，出错的比特组合可能使得余数为0，解决方法是严格挑选P，并使用位数较多的P

    基本链路控制机制

    • 掌握停-等协议、连续ARQ协议、滑动窗口、选择重传ARQ协议的工作原理

    停等协议 / 自动重传请求（ARQ）

    链路不会处于理想状态，需要差错控制和流量控制，一般情况下实用的停止等待协议会出现下面四种情况：

    1. 正常（不出差错）
    2. 数据帧可能出差错（但能被接收到）
    3. 数据帧可能丢失（不能被接收到）

      解决办法：超时重传

    4. 确认帧可能丢失或延迟（数据帧正确到达而发送端不知道）

      解决办法：每个数据帧都有一个序号

    连续ARQ协议

    发送完一个数据帧后，不用停下来等待确认帧，而是可以连续再发送若干个数据帧。

    • 结点A在每发送完一个数据帧时都要设置超时计时器。时间到则重传（对出错的帧和丢失的帧）
    • 发送的数据帧出错时，结点B有两种选择：
      • 向结点A发送否认帧，以便及早重传错误帧
      • 不作任何响应（不发送否认帧）。简单，使用较多

    信道传输质量差，误码率大时效率低

    滑动窗口

    若A一直没有收到B的确认信息，A不能无限制地发送数据帧，所以必须将已发送出去但未被确认的数据帧的数目加以限制（发送窗口）。这样序号位也可以循环重复使用，不会占用太多位数。

    对于发送窗口而言：

    • 只有发送窗口中的帧才能被发送 ——控制了流量
    • 只有收到了新的确认，窗口才会向前滑动 ——滑动窗口

    对于接收窗口而言：

    • 只有当收到的数据帧的发送序号落入接收窗口内才允许将该数据帧收下，在窗口外的帧则一律丢弃 ——解决了重复问题
    • 接收窗口WR>1时，接收端可以不按序接收
    • 接收端可以累计应答（确认）和捎带应答（确认）

    需要注意的是：

    1. n比特序号时，连续ARQ协议（WR=1）的发送窗口最大值应满足如下公式时才能正常工作： WT ≤ 2n-1
    2. 发送窗口越大，发送端需要的缓冲越大，对于停止等待协议， WT=1，则发送端的缓存空间只需能存放一个数据帧即可

    选择重传ARQ协议

    连续ARQ协议重传机制的缺陷：若连续数据帧的中间出现差错或超时，则从错误数据帧后的全部数据都必须重传。改进：选择重传ARQ协议，设法只重传出现差错的或超时的数据帧

    PPP协议概述

    但是实际生活中已经不怎么使用PPP协议了

    PPP是使用最多的链路层协议之一，是远程孤立计算机接入的主要协议，用于全双工、点对点链路。PPP协议的三个组成部分：

    • 封装PPP数据帧的方法
    • 链路控制协议 LCP ：用来实现建立、配置和释放数据链路连接

    • 网络控制协议 NCP ：支持不同的网络层协议，且每一种网络层协议都有一个对应的NCP

    PPP不提供使用序号和确认实现可靠传输，理由如下：

    • 在数据链路层出现差错的概率不大时，使用比较简单的 PPP 协议较为合理。
    • 在Internet环境下，PPP 的数据字段放入的数据是 IP 数据报。数据链路层的可靠传输并不能够保证网络层的传输也是可靠的。
    • 帧检验序列 FCS 字段可保证无差错接收。

    PPP协议的帧格式及透明传输

    关于透明传输问题，不同的传输方式不同：

    • 面向比特的同步传输：采用硬件完成零比特填充(同HDLC，在连续的5个1后插入1个0)
    • 面向字节的异步传输：特殊的字符填充法（字节填充法）

    PPP协议的工作过程

    PPPoE

    PPPoE (Point-to-Point Protocol Over Ethernet，或PPP Over Ethernet) ：“在以太网上运行PPP”，即将PPP帧再封装到以太网帧中来传输,以实现出传统以太网不能提供的身份验证、压缩等功能。

    局域网知识补充

    • 了解局网技术的发展和特点

    主要的局域网技术：以太网、令牌环网、令牌总线网、光纤分布式数据接口（FDDI）、异步传输模式（ATM）、 无线局域网

    局域网技术的特点：

    • 广播式信道：无连接、需要寻址、共享介质时需要介质访问控制
    • 误码率低：不需要复杂的可靠性控制，无编号、无确认

    介质访问控制（多路访问控制）：

    • 目的：确定每个节点能把信息送到通信介质上去的特定时刻
    • 分类1：
      • 载波侦听多路访问：以太网（CSMA/CD）、无线局域网（CSMA/CA）
      • 令牌传递访问控制：令牌环网、令牌总线网、FDDI
    • 分类2：
      • 共享式局域网：网中所有结点共享一条公共传输信道，如传统以太网、令牌网、FDDI
      • 交换式局域网：独占端口带宽，如ATM、交换式以太网

    以太网

    以太网的基础知识

    • 掌握以太网的种类、拓扑结构、服务方式、信道类型等
    • 理解合理分配信道的必要性

    以太网提供的服务：

    • 不可靠的交付，即尽最大努力的交付。
    • 有差错时就丢弃此帧，其他什么也不做，差错的纠正由高层来决定。
    • 以太网并不区分重传的帧，而是当作一个新的数据帧来发送。

    分类1：

    • 共享式以太网：是一种传统的以太网配置，它使用一条共享的传输介质（通常是电缆），将多台计算机连接到一个网络上。所有计算机都通过相同的传输介质进行通信，并使用一种称为"CSMA/CD"（载波侦听多路访问/冲突检测）的协议来管理数据的传输。

      集线器连接的以太网是共享式以太网。这是因为集线器本身没有智能的转发功能，它只是简单地将数据广播到所有连接的设备。缺点：一个站点发送的信号，可以被网上的其它所有站点收到，信息易被窃听。

    • 交换式以太网：是一种现代化的以太网配置，它使用交换机作为网络的核心设备。每台计算机都通过独立的连接与交换机相连，形成一个点对点的连接。交换机负责根据目标地址将数据包从一个端口转发到另一个端口，从而实现计算机之间的直接通信。

    传输介质：

    以太网规定：帧间最小间隔为 9.6 s，相当于 96 bit 的发送时间一个站在检测到总线空闲后，还要等待 9.6 s 才能再次发送数据。以太网不需要使用帧结束标志，也不需要考虑透明传输问题。

    中继器：

    一种在物理层上实现信号的放大与再生的网络设备，用以扩展局域网的跨度。集线器（HUB）是一种特殊的多端口中继器，所有连接端口共享网络带宽。

    集线器和交换机：

    宽带路由器和交换机的区别：

    1. 宽带路由器有WAN口和LAN口
    2. 当宽带路由器去掉WAN口时就是一个交换机，加上WAN口时就是一个网关

    以太网还采取一种叫做强化碰撞的措施。一旦检测到碰撞，除了立即停止发送顿外，还要再继续发送48比特的人为千扰信号(Jamming Signal),以便有足够多的位撞信号使所有站点都能检测出碰撞。

    导致冲突的两种情况：

    • 侦听到信道某一瞬时处于空闲状态时，两个以上的结点同时向信道发送数据。
    • 传播时延使得结点B已经发送的数据还没有传播到结点A，故而A侦听到信道是空闲的。

    争用期：

    在每个站发送数据刚刚开始的一个很短的可能发生冲突的时间间隔。也称为时间片、冲突域、冲突窗口。过了争用期，则不会再发生冲突。

    最短有效帧：

    帧的最小长度必须保证：在帧发送完之前，CSMA/CD协议能够检测到可能最晚到来的冲突信号。以太网规定：

    • 最短有效帧长为 64 字节，凡长度小于64字节的帧都是由于冲突而异常中止的无效帧。
    • 在接收端，长度不够64字节的帧都被认为是无效帧而丢弃。

    MAC地址/物理地址

    严格来说，MAC地址是对网络上各接口的唯一标识，而不是对网络上各设备的唯一标识。

    广播地址：FF-FF-FF-FF-FF-FF

    MAC帧

    这个MAC帧是以太网V2的MAC帧，和IEEE802.3MAC帧有细微差别，但现在已经被广泛采用。

    • 前同步码：10101010，使接收端在接收MAC帧时能够迅速实现比特同步
    • 帧开始标志：10101011
    • 只有目的地址才能使用广播地址和组播地址（多播地址）
    • 类型：类型字段的值是0x0800时，就表示上层使用的是IP协议。若类型字段的值为0x8137时，则表示上层使用的是Novell Netware系统的IPX协议。
    • 帧检验序列（FCS，Frame Check Sequence）使用循环冗余检验（CRC-32）

    以太网交换机工作原理总结：

    • 自学习：查找交换表中与收到帧的源地址有无相匹配的项目
      • 如没有，就在交换表中增加一个项目（源地址、进入的接口和有效时间）
      • 如有，则把原有的项目进行更新（进入的接口或有效时间）
    • 转发帧：查找交换表中与收到帧的目的地址有无相匹配的项目。
      • 如没有，则向所有其他接口（进入的接口除外）转发。（泛洪）
      • 如有，则按交换表中给出的接口进行转发

    生成树协议STP：

    不改变网络的实际拓扑，但在逻辑上切断某些链路，使得从一台主机到所有其他主机的路径是无环路的树状结构，从而消除了兜圈子现象。

    基础知识

    无线局域网按照传输技术可以分为三类：红外线局域网、蓝牙、802.11无线局域网（WIFI），前两者统称为无线个人区域网或个域网（WPAN）。

    为什么手机上是WLAN呢，因为不仅包含了WIFI还包含了WAPI，但是WAPI已经没用了。

    5G实际上是移动通信网络而WIFI是无线局域网，两者是不同的

    无线局域网的两种组网方式：

    • 基础设施模式：是最常见和常用的无线局域网组网方式。无线访问点（Wireless Access Point, AP）充当中心节点，负责无线信号的传输和管理。其他设备通过连接到访问点来获取无线网络接入。
    • 自组织模式：是一种去中心化的无线局域网组网方式。设备之间直接建立点对点的连接，形成一个临时的无线网络。每个设备既是终端节点，也是中继节点，负责数据的传输和转发。
