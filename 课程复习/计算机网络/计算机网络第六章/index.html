    域名系统DNS与DHCP

    • 了解分布式域名系统的概念
    • 掌握因特网的域名结构
    • 掌握域名的解析过程
    • 掌握DHCP协议的作用及其工作过程，包括DHCP中继

    分布式域名系统（Domain Name System，DNS）为用户提供域名与IP地址之间的映射。DNS系统是分布式的，这意味着没有单一的中央服务器负责所有域名的映射，DNS数据库分散存储在全球各地的多个DNS服务器上。

    域名结构

    常见的顶级域名：.com （公司和企业） .net （网络服务机构） .org （非赢利性组织） .edu （美国专用的教育机构） .gov （美国专用的政府部门） .mil （美国专用的军事部门） .int （国际组织）

    • 一个IP地址可以对应多个域名；当为了实现负载分担而使用多台计算机提供相同资源服务时，一个域名也可以对应多个IP地址
    • 域名中的标号都有英文和数字组成，每一个标号不超过63个字符(一般不会超过12个字符)，不分大小写
    • 级别低的域名写在左边,由多个标号组成的完整域名总共不超过255个字符
    • DNS既不规定一个域名需要包含多少个下级域名，也不规定每一级域名代表什么意思
    • 服务器的管辖范围不是以“域”为单位，而是以“区”为单位,每一个区设置相应的权限域名服务器

    四级域名的例子：www.whu.edu.cn 、 cas.whu.edu.cn 、 ehall.whu.edu.cn

    三级域名的例子：www.csdn.net 、 so.csdn.net 、 blog.csdn.net

    域名解析

    递归查询用的比较少

    • 递归查询：本地DNS服务器首先查询根域名服务器，然后查询顶级域名服务器，然后查询二级域名服务器，直到找到目标域名的IP地址。然后，本地DNS服务器将IP地址返回给DNS客户端，并将结果缓存。
    • 迭代查询：本地DNS服务器向根域名服务器、顶级域名服务器和二级域名服务器依次发出查询请求，然后将每个结果返回给DNS客户端。DNS客户端会根据这些信息继续向下查询，直到找到目标域名的IP地址。
    • 每个域名服务器都维护一个高速缓存，存放最近用过的名字以及从何处获得名字映射信息的记录。 可大大减轻根域名服务器的负荷

    DHCP

    DHCP（Dynamic Host Configuration Protocol）是一种网络协议，用于动态分配IP地址和其他网络配置信息给计算机和其他网络设备。

    其配置的主要内容包括：

    • IP地址范围（常称为“IP地址池”）及其子网掩码。
    • 租约期限：即客户端从DHCP服务器获得IP地址后的有效使用期限，超过租约期限后，客户端必须向服务器申请续订。DHCP也允许租约期限为“无限制” 。
    • 默认网关，如果客户端只与同一IP子网的计算机通信，则可以不配置此项。
    • 客户端使用的DNS服务器地址，如果不需要用域名来和其他计算机通信，则可以不配置此项。

    基础知识

    基础概念：

    • UA（User Agent，用户代理）：该客户端软件用来编写、显示、与本地邮件服务器的通信（发送/接受邮件）。如：Outlook Express, Foxmail等。
    • MTA（Message Transfer Agent，消息传输代理）：该服务器软件用来传递信件，向发件人报告邮件传送情况（已交付、被拒绝、丢失等）。如：Microsoft exchange Server，Sandmail等。

    SMTP、POP3、MIME协议：

    • 简单邮件传输协议（Simple Mail Transport Protocol，SMTP）：用一种用于发送电子邮件的协议，通常使用TCP/IP协议的25号端口，是一种无状态协议，它不会保留关于会话的信息。
    • MIME多用途因特网邮件扩展协议：用于扩展标准电子邮件格式的协议，它允许电子邮件传输不仅包括纯文本信息，还可以包括各种多媒体内容，如图像、音频、视频和附件等。
    • 5 个新的邮件首部字段，它们可包含在[RFC 822]首部中。这些字段提供了有关邮件主体的信息。
    • 定义了传送编码，可对任何内容格式进行转换，而不会被邮件系统改变。
    • 定义了邮件内容的格式，对多媒体电子邮件的表示方法进行了标准化。

    MIME主要包括以上三个内容，由于不是重点，这里不再赘述。

    • 邮局协议（Post Office Protocol，POP）：一种用于检索电子邮件的协议，通常使用TCP/IP协议的110号端口，是一种有状态协议，它跟踪每个电子邮件客户端的会话状态，包括已读和未读邮件。

    文件传输

    • 掌握FTP协议的特点及工作方式
    • 了解TFTP协议特点与工作方式

    FTP协议

    • 基于客户端-服务器模型：FTP工作在客户端-服务器架构上，用户需要一个FTP客户端软件来连接到FTP服务器。
    • 双通道控制：FTP使用两个独立的通道进行通信——控制通道（用于发送命令）和数据通道（用于传输文件数据）。
    • 明文认证：传统的FTP协议在发送用户名和密码时不提供加密可能被截获。
    • 被动和主动模式：FTP有两种连接模式，被动（PASV）和主动（PORT）。这两种模式处理数据连接的建立方式不同。
    • 支持匿名访问：FTP服务器可以配置为允许匿名用户登录，通常用于公共文件的下载。
    • 适用于大文件传输：FTP是传输大文件的理想选择，因为它支持断点续传和批量处理。

    TFTP协议

    特点：

    • 简化的设计：TFTP设计简单，实现容易，适用于小型文件的快速传输。不适合大型文件或高安全需求，由于其设计简单和使用UDP，TFTP不适合传输大文件或在安全要求较高的环境中使用。
    • 基于UDP：不同于FTP协议使用TCP，TFTP使用UDP（用户数据报协议）进行数据传输，这使得它更轻量级但也不提供内置的可靠性保证。
    • 无需认证：TFTP不提供身份验证机制，任何请求的客户端都可以从TFTP服务器下载或上传文件。
    • 无连接状态：由于基于UDP，TFTP不维护一个持久的连接状态，每个数据包都是独立传输的。
    • 错误恢复简单：TFTP使用简单的错误恢复机制，通常是重传丢失的数据包。

    工作方式：

    • 读取请求（RRQ）：客户端向服务器发送一个读取请求来下载文件。请求中包含文件名和传输模式。
    • 写入请求（WRQ）：客户端发送一个写入请求来上传文件到服务器。
    • 数据传输：数据以512字节的数据包形式传输。每个数据包都包含一个序号，接收方需要对每个数据包进行确认（ACK）。如果数据包丢失或出错，发送方会重传该数据包。
    • 传输结束：当一个小于512字节的数据包被发送时，表示文件传输结束。客户端或服务器接收到最后一个数据包后，传输过程完成。

    基础知识

    • 万维网：是基于超文本和超媒体构建的全球信息空间。WWW（World Wide Web）简称Web，又称万维网或环球网，是一种非常方便、快捷的信息浏览服务。

    • 超文本：是一种文本组织方式，允许文本到文本的链接。

    • 超媒体：是超文本的扩展，包含了文本以外的其他媒体元素和链接。

    • URL的格式：

      • 协议：指明了用于访问资源的协议类型，如 http、https、ftp 等。
      • 主机名：指定托管资源的服务器的域名或IP地址。
      • 端口（可选）：服务器监听请求的端口号。如果省略，则使用协议的默认端口。
      • 路径：资源在服务器上的具体位置。通常指文件或目录的路径。

    远程登录

    • 了解Telnet协议及使用方式

    基础介绍

    用户通过Telnet客户端软件连接到服务器。用户的键入就直接发送到服务器，服务器端的输出也会反馈到用户的终端。主要用于远程管理服务器、调试网络服务和运行文本基础的应用程序。

    • 由于安全性较低，所有通过Telnet发送的数据，包括用户名和密码，都是以明文形式传输的，容易被拦截和监听，现在它已经逐渐被更安全的协议如SSH所取代。
    • Telnet协议定义了一个通用终端数据标准 — NVT（Net Virtual Terminal，网络虚拟终端）结构。NVT使Telnet能够适应各种计算机和操作系统的差异。

    简单网络管理协议SNMP

    • 了解网络管理的基本概念和简单网络管理协议SNMP

    网络管理的基本概念

    网络管理：即对网络的运行状态进行监测和控制，使其能够有效、可靠、安全、经济地提供服务。管理对象包括：硬件资源（各种网络互连设备、计算机及其外围设备、接口设备和物理介质）软件资源（通信软件（实现通信协议的软件）、操作系统以及各种应用软件）

    SNMP

    SNMP是TCP/IP协议族的一个应用层协议，工作在无连接的UDP协议之上，使用161 （SNMP Agent使用）和162 （SNMP Manager使用）两个熟知端口，SNMP消息采用抽象语法表示ASN.1（不依赖于任何硬件系统的抽象性描述语言，有利于实现网络不同系统之间的无缝连接和通信）进行编码。

    SNMP管理器从SNMP代理收集数据有两种方法：

    • 轮询：SNMP管理器周期性地向被管对象的网管代理发送轮询请求（请求MIB中的变量信息），代理收到请求后回复响应，管理器再根据响应进行处理。

    • 事件驱动：当被管对象某状态变量超出阈值时，代理则实时发送异常事件通知（trap）给管理器，而不必等到管理器轮询到它的时候才报告这些错误信息。

    • 常用命令：如ping，tracert/tracerout、netstat等命令可看成是系统提供的简单TCP/IP网络管理工具。

    P2P应用

    • 了解P2P应用的工作机制

    与传统的客户端-服务器模型不同，P2P网络中没有专用的服务器，网络功能是由网络中的所有节点共同完成的，其有如下特点：

    • 节点间直接通信：在P2P网络中，节点之间直接交换数据，无需通过中央服务器。
    • 资源共享：每个节点可以提供和接收资源，如文件共享、数据存储或计算能力。
    • 动态网络：网络的组成随时在变化，因为节点可以随时加入或离开网络。
    • 自我组织：P2P网络能够自动配置和管理自己，适应网络变化和节点的加入或退出。
