    OS如何控制

    OS可以通过shell进行控制，只要将OS启动之后的第一个程序交给shell就行了，下面介绍一下如何实现这个过程。

    第一步：进入INIT( )

    前面的OS运行流程是一样的，我就不赘述了，值得注意的是，内核主函数会将INIT进程放在进程队列的最前面，然后进入restart( )函数启动进程表第一个进程，也就进入了INIT进程的运行了。

    第二步：进入shabby_shell( )

    INIT进程会通过fork( )创建多个进程（多少个TTY就有多少个），每个进程都执行shabby_shell( )即简陋的命令行，至此OS可以成功交给shell控制了，shell的功能全封装在inst.tar中了。

    注意：根据书上的代码这里总共有三个TTY，TTY0执行完之后就没什么用了，TTY1和TTY2都是shell，只要shell不执行完毕，就会一直运行。

    进入了shell之后，就可以通过shell控制整个操作系统啦！

    shell如何工作

    根据已经完成的任务三（在同一个 shell 中，支持多任务执行），可以总结出shell的工作流程，这里的shell支持多任务：

    1. 打开终端设备文件：使用给定的tty_name参数打开终端设备文件两次，分别用于标准输入和标准输出
    2. 进入循环：等待用户输入命令
    3. 读取用户输入：
      1. 在每次循环开始时，输出命令提示符$ 。
      2. 每次最多读取70个字符，并将其存储在rdbuf缓冲区中
    4. 解析命令：
      1. 根据空格和空字符将用户输入的命令解析成多个参数，存储在argv数组中。
      2. 通过循环遍历rdbuf缓冲区中的字符，将非空格且不为0的字符作为一个参数的起始点，将空格或空字符作为一个参数的结束点，并将起始点和结束点之间的字符存储在argv数组中。
      3. 当解析完所有参数后，将argv数组的最后一个元素设置为NULL，表示参数列表的结束。
    5. 处理多个命令：
      1. 定义了一个二维数组multi_argv，用于存储多个命令的参数列表。定义了变量num_proc表示命令的数量，变量sec_count表示当前命令的参数数量。定义了变量error表示是否发生错误。
      2. 遍历argv数组中的参数，对于每个参数：
      3. 如果不是"&"字符，将其添加到当前命令的参数列表multi_argv[num_proc - 1]中。如果是"&"字符，表示当前命令的参数列表结束，将其置为NULL，然后更新num_proc和sec_count，开始处理下一个命令。
      4. 如果num_proc超过了MAX_SHELL_PROC的限制，将error置为1，输出错误提示信息"Too many commands!"。
    6. 执行命令：
      1. 如果没有发生错误（error为0），对每个命令执行以下操作：使用open函数打开第一个参数作为文件，并检查返回值。如果返回值为-1，表示文件打开失败，则输出错误信息。如果返回值不为-1，表示文件打开成功，则关闭该文件描述符。
      2. 使用fork函数创建子进程。如果返回值不为0，表示当前为父进程，则使用wait函数等待子进程的结束。如果返回值为0，表示当前为子进程，则使用execv函数执行指定命令。
      3. 执行完所有命令后，回到循环的开头，等待下一次用户输入。
    7. 关闭文件描述符：在循环结束后，使用close函数关闭标准输入和标准输出的文件描述符。
